# Лабораторная работа
## 3.2 Метод Зейделя

## Цель
Реализовать метод Зейделя для решения системы \(Ax=b\) c помощью циклов.

## Реализация
Реализована функция `zeidelMethod(A, b, max_iter, tol)`, которая:
- принимает `A`, `b`, `max_iter`, `tol`
- выполняет итерации метода
- на каждой итерации считает невязку \(\|Ax-b\|\)
- останавливается при \(\|Ax-b\| < tol\)
- возвращает приближённое решение и массив невязок

Также реализована генерация тестовой положительно определенной матрицы:
\[
A = R^T R + nI,
\]
где \(n > 100\).

## Параметры эксперимента
- Размер матрицы: `n = 120`
- Максимум итераций: `max_iter = 10000`
- Точность: `tol = 1e-8`
- Случайный сид: `seed = 67`

## Результаты
- Число итераций: `24`
- Точная ошибка: `4.1680109896e-11` (мала, то есть метод находит ответ близкий к реальности)
- Время `numpy.linalg.solve` имеет порядок `0.0003` секунд
- Время метода Зейделя:  
  - первый запуск: `0.2443` с  
  - последующие запуски были уже быстрее (скорее всгео за счет numba), где то примерно `0.0016` с

## Вывод
Метод сошёлся за 24 итерации, точная ошибка очень мала (\(\sim 10^{-11}\)).  
Разница во времени между первым и следующими запусками связана с `numba`. Первый запуск включает JIT-компиляцию, последующие работают уже на скомпилированном коде.  
Для матрицы размера \(120\) `numpy.linalg.solve` всё равно быстрее, так как это очень оптимизированная библиотечная реализация.
